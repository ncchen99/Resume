# 第 4 頁|個人專案:Tuckin 🥕

## 雙平臺聚餐媒合 App|從 0 到 1 獨立完成,上線累積 300 位使用者

**GitHub:https://github.com/ncchen99/Tuckin**
**上架:App Store(iOS)+ Google Play(Android)**

📷 截圖預留位:App 主畫面、預約流程、餐廳投票畫面(可用 repo docs/images 中的官方截圖)

### 專案簡介

Tuckin 是專為成大學生設計的聚餐媒合平臺:使用者預約下週聚餐時段,系統自動分組(3 人以上成團),再由成員投票決定餐廳,讓陌生的同學透過一頓晚餐自然建立連結。從產品發想、UI 設計(Clay 風格設計語言)、前後端開發到雙平臺上架,全程由我獨立完成。

### 解決誰的問題

大學生想認識新朋友,但主動社交的門檻很高。Tuckin 把「認識陌生人」的阻力降到最低——你只需要按下預約,剩下的分組、投票、時間協調都由系統完成。平臺正式上線後即累積 300 位使用者,許多使用者因此養成了「每週和新朋友吃一頓晚餐」的社交習慣,把偶然的相遇變成日常的期待。

### 系統與技術亮點

- **Real-time 即時系統**:聚餐狀態(分組中、投票中、確認出席)即時同步到所有成員裝置,搭配 Firebase Cloud Messaging 推播提醒。
- **身分信任機制**:Google OAuth 登入,成大 email 自動獲得學生認證,可選擇僅與校內同學分組,兼顧安全與信任。
- **分組演算法**:依時段、飲食偏好與個性測驗結果自動分組,並以出席率停權機制維護聚餐品質。
- **後端架構**:Supabase(PostgreSQL + Auth + RLS)作為核心資料層,FastAPI 處理分組與排程邏輯。

### 技術棧

Flutter(Dart)、Supabase(PostgreSQL + Auth)、Google OAuth、FastAPI(Python)、Firebase Cloud Messaging、PLpgSQL

# 第 6 頁|網頁 App:Coffee Pocket ☕

## 整合大臺南咖啡廳資訊的 LLM 語意搜尋平臺

**GitHub:https://github.com/ncchen99/Coffee-Pocket**
**線上網址:https://tainan-cafe.web.app/**

📷 截圖預留位:桌面版三欄地圖介面、手機版搜尋與標籤篩選畫面(repo docs/assets 有現成 GIF)

### 專案簡介

Coffee Pocket 是以臺南咖啡廳為主題的探索與收藏工具。使用者可以用自然語言(「有插座適合工作的咖啡廳」)或實用標籤(插座、不限時、適合讀書)快速找到符合情境的店家,並收藏進個人口袋名單。支援桌面與行動裝置,可安裝為 PWA。

### 解決誰的問題

在臺南找一間「有插座、不限時、適合工作」的咖啡廳,往往要翻大量評論才能確認。Coffee Pocket 把散落各處的資訊結構化,讓「找店」從反覆爬文十幾分鐘變成一次搜尋幾秒鐘——改變了學生與遠端工作者挑咖啡廳的習慣。

### LLM 資料 Pipeline(技術核心)

- **多來源資料匯整**:從 Cafe Nomad、Google 地圖清單、Instagram 貼文等多個來源抓取店家資料,經 Google Places API 取得基本資訊,並以 Place ID 比對去重。
- **LLM 語意萃取**:蒐集公開評論後,透過 LLM(OpenRouter / OpenAI API)萃取為結構化語意標籤(如「插座多」「適合讀書」),每個標籤附帶信心分數與證據來源數量(例:「插座多 — 12 則評論提及」)。
- **自然語言查詢**:使用者輸入的口語需求由 Supabase Edge Function 解析語意,轉換為標籤篩選條件,支援 AND / OR 組合。
- **社群校正機制**:登入使用者可對標籤投票(贊同/不贊同),持續校正資料正確性。

### 技術棧

前端:React 18、TypeScript、Vite、Tailwind CSS + daisyUI、Mapbox GL JS、React Query、PWA
後端:Supabase(PostgreSQL + PostGIS + Auth + Edge Functions)、FastAPI
資料處理:Python 3.11、Playwright、LLM(OpenRouter / OpenAI)
部署:Firebase Hosting、Fly.io、Cloudflare R2

## 即時同步的分帳 PWA|獨創個人與團體雙軌分帳邏輯

**GitHub:https://github.com/ncchen99/Zplit**
**線上網址:https://zplit.web.app**

### 專案簡介

Zplit 是一款行動優先的分帳 Web App(PWA),以 Firebase 為核心實現跨裝置即時同步,讓使用者在群組活動與一對一往來中都能快速記帳、即時計算誰該付誰。

### 解決誰的問題與獨創設計

市面上的分帳工具多以「單次活動群組」為中心,但真實生活的帳務往往同時存在兩種型態:一起出遊的「團體帳」,以及和室友、好友之間的長期「個人往來帳」。Zplit 採用獨創的**個人與團體雙軌分帳邏輯**:

- **群組分帳**:多人活動費用拆分,支援平均、指定金額、百分比三種分攤模式,自動計算每位成員淨額並給出最少轉帳次數的建議結算路徑。
- **個人往來**:獨立管理你與每位聯絡人的借貸/代墊紀錄,檢視淨額與歷史互動。

這讓固定朋友圈可以長期使用同一套工具,不必為每次聚會另建群組,「先幫忙墊、之後一起算」成為無壓力的日常習慣。

### 技術棧

React 19、TypeScript、Vite、Tailwind CSS 4 + daisyUI 5、Zustand、Firebase(Auth + Firestore 即時同步)、react-i18next(繁中/英文雙語)、Playwright、Cloudflare R2 / Turnstile

# 第 9 頁|競賽作品:捷境 MetroSense 🚇

## 北捷黑客松 第一名 + 宏碁特別獎

**GitHub:https://github.com/ncchen99/Metro-Sense**
**線上網址:https://metro-sense.vercel.app**

📷 截圖預留位:App 主畫面、語音問答介面(repo docs/images 有 demo 圖)

### 專案簡介

捷境 MetroSense 是結合 AI 的智能交通助手,專為台北捷運設計。核心概念是針對不同族群需求的「個性化開關」:區分**通勤模式**(專注通勤效率)與**旅遊模式**(提供景點推薦),為使用者打造更簡潔、專注的介面。此作品獲得北捷黑客松第一名及宏碁特別獎。

### 解決誰的問題

通勤族要的是「最快到公司」,觀光客要的是「沿途有什麼好玩」——同一套捷運 App 很難同時服務兩者。MetroSense 以模式切換讓介面隨使用情境改變,並以自然語言與語音互動取代層層選單,讓長輩與觀光客也能無門檻地查詢路線。

### 系統與技術亮點

- **語音互動系統**:語音轉文字(STT,支援多種音訊格式)→ AI 問答 → 文字轉語音(TTS)完整閉環。
- **SSE 即時串流**:AI 回應以 Server-Sent Events 逐字輸出,對話體驗流暢。
- **高可用架構**:前端部署 Vercel,後端主力跑在 Fly.io(容器化、冷啟動 < 1 秒),並以 Render 作為備援——前端會在 1 秒內自動偵測主服務可用性並無縫切換,達成使用者無感知的自動容錯。

### 技術棧

React 18、Vite、Tailwind CSS、Framer Motion、FastAPI、OpenAI API(GPT-4 / STT / TTS)、SSE、Vercel + Fly.io + Render(容錯切換)

## InkSync|分散式多裝置 E-Paper 控制系統

**GitHub:https://github.com/ncchen99/InkSync**

📷 截圖預留位:電子紙實體照片、Flutter App 控制介面(repo docs/images 有實拍圖)

### 專案簡介

InkSync 是一套軟硬整合的 IoT 系統:用手機 App 同時控制多台電子紙裝置的顯示內容,適用於展場看板、桌面資訊牌等情境。

### 系統架構與技術亮點

- **MQTT 分散式架構**:ESP32-S3 韌體與 Flutter App 均為 MQTT Client,桌機 Mosquitto 擔任 Broker;每台裝置以 MAC 位址對應專屬 Topic(`devices/{MAC}/cmd` / `state`),可無上限擴充裝置數量。
- **影像處理管線**:App 端裁切圖片後上傳 Cloudflare R2 取得公開 URL,韌體端下載後進行 JPEG/PNG 解碼與 dithering 演算法,輸出至 4.0 吋 7 色電子紙。
- **即時狀態回報**:App 訂閱裝置狀態 Topic,即時顯示 queued / downloading / success / error。
- **裝置綁定體驗**:支援手動輸入或掃描 MAC 條碼綁定裝置(mobile_scanner + mDNS)。

### 技術棧

硬體:ESP32-S3、Waveshare 4.03" 7-Color E-Paper|韌體:PlatformIO、PubSubClient、ArduinoJson、LittleFS|App:Flutter、Riverpod、mqtt_client|基礎設施:Mosquitto MQTT Broker、Cloudflare R2
